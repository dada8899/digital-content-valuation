## 1. 千川投手的完整一天（时间线重建）

### 1.1 逐小时还原：月消耗30万腰部美妆品牌投手

**07:00-08:00 | 通勤 + 刷手机**
- 刷到几条竞品广告，截图丢到工作群："这个素材在跑，注意一下"
- 顺手看一眼千川App的推送通知——昨晚计划是否有异常
- 心理状态：半焦虑半惯性。脑子里已经在算"昨天消耗跑了多少"

**08:30-09:30 | 开后台，看昨天数据**
- 打开千川后台，第一眼看消耗和ROI两个数字
- 如果昨天整体消耗正常、ROI在2以上——松一口气，今天可以不那么紧张
- 如果消耗掉了或ROI崩了——整个人的状态立刻进入"救火模式"
- 逐条检查在跑计划的状态：哪些还在消耗，哪些掉速了，哪些死了
- 标记"今天要关掉的计划"和"今天要调整的计划"
- **此时不需要外部工具，完全在千川后台内操作**

**09:30-10:30 | 素材盘点 + 催素材**
- 打开素材库，清点"还在跑"的素材数量——通常5-8条
- 算一下本周的"素材缺口"：需要15-20条保持消耗，现有可用5-8条，缺口7-12条
- 在微信/飞书群里催剪辑："今天能交几条？""昨天说好的3条呢？"
- 如果剪辑回复"还在做/还没想好方向"——焦虑升级
- **这是一天中"素材焦虑"第一次浮现的时刻**

**10:00-11:30 | 找素材方向（核心痛点时段）**
- 打开有米有数/蝉妈妈，看竞品最近在投什么
- 打开巨量创意，看行业跑量榜
- 截图3-5条"看起来不错"的竞品素材，发给剪辑说"往这个方向做"
- 剪辑问："你要什么感觉的？"投手说不清楚，只能说"就这种感觉"
- **这个环节耗时1-1.5小时，产出的是"模糊的方向"而不是"可执行的方案"**
- **如果有一个工具能在这个时刻告诉他"这条素材在千川ROAS 2.5，品类匹配你，500块买断"——他大概率会认真看**

**11:30-12:00 | 调整在跑计划**
- 调出价：掉量的计划提价5-10%试试能不能恢复
- 调定向：放宽或收紧人群包
- 复制跑量计划：把跑得好的计划复制2-3个，微调出价
- **这些操作的实际效果有限（素材才是核心杠杆），但不做就是"摸鱼"**

**12:00-13:00 | 午饭**
- 边吃饭边刷投手群的消息
- 群里有人晒跑量数据，有人吐槽素材不行
- 偶尔看到有人在群里分享"最近跑得好的素材类型"

**13:00-15:00 | 新素材上线测试（关键动作）**
- 剪辑交了3-5条新素材
- 每条素材建2-3个计划（不同出价/不同定向），给500-2000元测试预算
- 上传 → 过审（等待20分钟到2小时） → 开投
- 前2小时盯着数据看：有没有起量、CTR多少、转化如何
- 10条里通常7-8条在2小时内"被判死"——系统给了少量曝光但CTR/CVR不达标，自动降速
- **每条测试失败 = 白花500-2000元。这个成本是投手买外部素材的心理锚点——"外面买一条500块，如果跑量概率比自测高一倍，就值了"**

**15:00-17:00 | 继续盯数据 + 微调**
- 对跑起来的计划：放预算、提预算上限
- 对半死不活的计划：观察一阵子，要么救活（换定向/提出价）要么放弃
- 对新素材表现好的：立刻让剪辑按这个方向多做几条变体
- **这是一天中最"机械化"的时段，大量重复操作**

**17:00-18:00 | 写日报 + 复盘**
- 整理今天的消耗、ROI、素材测试结果
- 写一份给老板/主管看的日报
- 核心数据：今日消耗XX万，ROI X.X，测试X条素材，跑起来X条
- **最怕的日报内容："素材全部扑街，消耗下滑，ROI不及预期"**
- 规划明天的素材需求，再催一遍剪辑

**18:00-19:00 | 下班（但不真的下班）**
- 千川计划还在跑，晚间流量高峰还没到
- 手机上随时看数据

**19:00-22:00 | 晚间监控**
- 晚间是抖音用户活跃高峰，也是广告竞价最激烈的时段
- 需要关注跑量计划是否稳定、是否有突然掉量的情况
- 如果数据好：安心，可以做别的事
- 如果数据差：焦虑，可能再调整一轮出价

**22:00-23:00 | 看一下最终数据，设好凌晨出价策略**
- 调低凌晨出价（凌晨竞价松但人群质量差）
- 看一眼今天总消耗和ROI
- **如果今天有素材跑起来了——心情不错，明天有弹药了**
- **如果今天全军覆没——带着焦虑入睡，想着明天从哪搞新素材**

### 1.2 工具切入点分析

| 时段 | 投手在做什么 | 工具切入可行性 | 理由 |
|------|------------|-------------|------|
| 08:30-09:30 看数据 | 千川后台 | 低 | 他在看自己账户的数据，不需要外部工具 |
| **09:30-10:30 素材盘点+催素材** | **微信群 + 焦虑** | **中** | 他意识到"素材不够了"，但此时还没有"买素材"的心智，只会催自己团队 |
| **10:00-11:30 找素材方向** | **有米有数/蝉妈妈/巨量创意** | **最高** | **这是他唯一一次"主动去外部寻找素材信息"的时段。如果我们的工具能在这个环节提供"带数据的可直接购买的素材"，替代了"看竞品→截图→发给剪辑→等剪辑翻拍"的低效链路，价值最明显** |
| 13:00-15:00 测试新素材 | 千川后台 | 中 | 如果他手里有从平台买的素材，这个时段会用；但切入点在上一个环节 |
| 17:00-18:00 写日报 | 文档工具 | 低 | 不相关 |
| 22:00-23:00 复盘 | 手机 | 低 | 太晚了，不会做采购决策 |

**结论：工具应该插入的环节是每天10:00-11:30的"找素材方向"时段。**

理由：
1. 这是投手每天唯一一次主动打开外部工具（有米有数/蝉妈妈）的时刻
2. 他此时的目标是"找到跑量方向"，而我们能提供的是"带数据验证的方向+可直接购买的素材"
3. 如果工具做成千川后台的插件/功能模块，他连切换工具都不用——在千川后台内就能完成"看数据→找素材→购买→上传测试"的闭环
4. 从"看到竞品素材"到"翻拍出来"通常要2-3天，而"直接购买素材"可以让"今天发现→今天上线测试"成为可能，节省的是时间

**但有一个前置条件：他必须先有"可以买别人跑量素材"这个认知。** 目前99%的投手没有这个心智模型。所以第一批用户必须靠BD手动教育，不能靠自然发现。

## 2. "购买决策"的微观分析

### 2.1 从"看到这个工具"到"花500块买一条素材"的完整心理关卡

#### 第一关：认知关（"这是什么？"）
- **场景**：投手在千川后台/有米有数/投手群中，看到一个"素材推荐"的入口或广告
- **心理**："又一个卖素材的？""跟代拍机构有什么区别？"
- **关键信息需求**：一句话说清楚"我们跟你现在用的有什么不同"
- **有效话术**："带千川真实跑量数据的素材，别人账户ROAS 2.5的素材你可以直接买来投"
- **无效话术**："AI驱动的内容交易市场""创作者经济新范式"
- **通过率估算**：看到入口的人中，约20-30%会点进去看看。大部分人看到"又一个素材工具"就略过了
- **通过关键**：必须在第一屏（3秒内）传递"带跑量数据"这个差异化信息

#### 第二关：兴趣关（"跟我有关吗？"）
- **场景**：点进产品页面/小程序，看到素材列表
- **心理**："有没有我这个品类的？""数据是真的吗？"
- **关键行为**：搜索自己的品类（如"面膜""精华"），看有没有匹配的素材
- **关键转折点**：
  - 如果搜到了品类匹配、数据看起来不错的素材 → 继续往下看
  - 如果搜不到品类匹配的 → 直接离开，认为"跟我没关系"
  - 如果数据看起来太假/太完美 → 直接离开，认为"骗人的"
- **通过率估算**：进入页面的人中，约40-50%能找到品类匹配的素材并产生兴趣
- **通过关键**：品类覆盖率要够（至少覆盖美妆、食品、3C这三大品类），数据展示要真实可信（不能只显示ROAS，要显示消耗额、投放天数、品类标签等上下文信息）

#### 第三关：信任关（"数据是真的吗？"）
- **场景**：看到一条素材标注"近7天千川ROAS 2.5，消耗8万"
- **心理**："这个数据谁验证的？""会不会是商家自己编的？"
- **关键疑虑**：
  1. 数据来源——是千川后台截图、API对接、还是自报？
  2. 数据时效——是昨天的还是上个月的？
  3. 数据真实性——有没有刷数据的可能？
- **信任建立方式（按有效性排序）**：
  1. 平台直连千川API展示数据（最高信任）→ 但千川不会给第三方API
  2. 经过验证的千川后台截图+第三方审计 → 可行但繁琐
  3. 成功案例展示（"XX商家买了这条素材，在自己账户跑出ROAS 2.0"）→ 最现实的方式
  4. 同行推荐（"群里的XX说用了不错"）→ 冷启动后最有力的信任来源
- **通过率估算**：有兴趣的人中，约30-40%能建立初步信任。剩下的人卡在"不确定数据真假"
- **通过关键**：信任建立不可能靠产品功能解决，只能靠案例和口碑。前50个商家必须手动服务，用他们的真实结果建立信任

#### 第四关：价格关（"值不值？"）
- **场景**：决定要不要花500块买这条素材
- **心理账本计算**：
  - "自己测一条素材的成本：500-2000元（测试预算）+300元（制作成本）= 800-2300元"
  - "买这条的成本：500元。如果跑量概率比自测高一倍（从5%到10%），那净省了至少300-1500元"
  - "但如果跑不动，500块就打水漂了——跟我自己测没区别"
- **心理价格锚点**：
  - 500元以下 = "无脑买，试试呗"（决策时间<1天）
  - 500-1000元 = "看看数据再决定"（决策时间1-3天）
  - 1000-2000元 = "得请示主管"或"得跟团队商量"（决策时间1-2周）
  - 2000元以上 = "这个价格我还不如找达人"（基本不买）
- **通过率估算**：建立信任后，约60-70%的人在500元以下价位会购买。1000元以上急剧下降
- **通过关键**：定价必须卡在500-800元的"甜蜜区"。试用机制（先免费试1条）能大幅提高通过率

#### 第五关：行动关（"怎么操作？"）
- **场景**：决定买了，开始操作
- **流程要求**：
  1. 选素材 → 下单支付 → 获取源文件 → 上传到千川后台 → 建计划投放
  2. 这个流程每一步都不能卡
- **最容易卡住的地方**：
  - 支付环节：如果需要注册账号、绑定银行卡、走企业采购流程 → 放弃
  - 文件获取：如果下载慢、格式不兼容千川 → 放弃
  - 授权文件：如果需要单独上传授权证明到千川"投放资质"里 → 嫌麻烦但能接受
- **通过率估算**：决定购买的人中，约80-90%能完成操作（前提是流程足够简洁）
- **通过关键**：支付必须支持微信/支付宝个人支付（不走对公）；源文件必须是千川能直接识别的格式（MP4，1080P）；授权文件平台自动生成

#### 第六关：复购关（"下次还买吗？"）
- **场景**：买了一条素材，投了千川，看结果
- **三种结果及对应行为**：
  1. **跑量了（ROAS 1.5+）**→ "这个平台靠谱！"→ 下周再来买 → 推荐给同行
  2. **数据一般（ROAS 0.8-1.5）**→ "跟自己做差不多"→ 可能再试一两次 → 如果连续一般则放弃
  3. **完全没跑起来**→ "就知道不靠谱"→ 不会再买 → 可能在群里吐槽
- **复购率估算**：
  - 如果首次购买跑量成功率>40% → 复购率约50-60%
  - 如果首次购买跑量成功率20-40% → 复购率约20-30%
  - 如果首次购买跑量成功率<20% → 复购率<10%，产品基本失败
- **通过关键**：首次购买的体验决定一切。前50个商家必须手动筛选最优质的素材给他们，确保首次成功率尽可能高。这不是"作弊"，是冷启动的必然策略

### 2.2 全链路转化漏斗

```
看到入口/广告：        1000人
  ↓（点击率25%）
进入产品页：           250人
  ↓（品类匹配+兴趣45%）
产生兴趣浏览素材：     112人
  ↓（建立信任35%）
信任并有购买意向：     39人
  ↓（价格可接受65%）
决定购买：             25人
  ↓（完成支付85%）
实际购买：             21人
  ↓（首次体验后复购35%）
成为回头客：           7人
```

**从曝光到回头客的转化率：约0.7%。** 这意味着如果要获得100个回头客，大约需要曝光14,000个潜在用户。

**关键瓶颈**：信任关（只有35%通过）和复购关（只有35%回来）。这两个环节加在一起的损耗率约为88%——也就是说，即使用户对价格和品类都满意，因为"不确定数据真假"和"首次体验不够好"，近九成人会流失。

**破局点**：不靠漏斗优化，靠BD手动推荐。前100个商家不应该走自助购买流程，而是一对一BD沟通，手动撮合最匹配的素材，确保首次成功率。等积累了30-50个成功案例后，再开放自助购买。

## 3. 创作者的"第一次体验"设计

### 3.1 从"第一次听说"到"完成上架"的完整链路

**第一次听说（触达渠道）**
- 最可能的触达方式：微信私信/小红书私信（BD主动联系）
- 次可能：创作者群里有人分享"我在XX平台卖旧视频赚了XXX"
- 最不可能：创作者自己搜索到（没有这个心智）
- **关键信息**：一句话讲清楚——"你有X条发布超过14天的视频，可以授权给商家投千川，每条能赚200-800元"

**打开产品（前10秒）**
- 形态：微信小程序（不是App，不是网页）
- 打开后第一屏必须展示：
  1. "你有32条内容可以上架"（具体数字，从抖音主页抓取）
  2. "预估总收益：1,200-3,600元"（给一个范围，不给精确数字）
  3. "一键上架"按钮
- **如果第一屏看到的是注册表单/品牌介绍/功能说明 → 80%的人关掉**

**前30秒（关键留存窗口）**
- 用户行为：扫一眼自己的内容列表，看看估价
- 心理活动："这条值300？还行""这条只值80？太少了""整体能赚1000多？试试呗"
- **最可能放弃的原因**：
  1. "要绑定抖音账号？我不太敢"（隐私/安全顾虑）→ 必须说清楚只读取公开信息
  2. "估价太低，不值得折腾"（预期管理失败）→ 估价要稍偏高而不是偏低
  3. "流程太复杂，要填这么多东西"（操作门槛）→ 一键上架，不要任何多余步骤

**1-3分钟（决策窗口）**
- 如果留下来了，用户会：
  1. 点几条内容看看详情（估价依据、会被怎么用）
  2. 看"授权协议"摘要（关心"会不会被滥用""能不能撤销"）
  3. 决定是否上架
- **需要回答的三个核心问题**：
  1. "卖了之后我的视频会被怎么用？" → 回答："只能用于千川广告投放，不能发自然流量"
  2. "我随时可以下架吗？" → 回答："可以，一键下架，已售出的授权到期后自动终止"
  3. "钱怎么到我账上？" → 回答："售出后24小时内到账微信零钱"

**3-5分钟（完成上架）**
- 操作流程必须是：
  1. 选择要上架的内容（默认全选发布14天以上的，可手动取消）
  2. 确认授权范围（千川广告投放，不可修改原始内容 / 可混剪 等选项）
  3. 确认价格（平台建议价，可自定义但不建议）
  4. 点"确认上架"
- **5分钟内完成的前置条件**：
  1. 不需要下载/上传视频（平台从抖音公开页面抓取）
  2. 不需要签合同（电子协议一键同意）
  3. 不需要实名认证（首次可延后到提现环节）
  4. 不需要设置定价（平台给建议价，一键接受）

### 3.2 "在哪一步最可能放弃"分析

| 步骤 | 流失率 | 主要流失原因 | 对策 |
|------|--------|------------|------|
| 收到私信 → 点开 | 60-70% | "不感兴趣""以为是骗子""没时间看" | 话术要具体到数字（"你有32条内容可赚1200元"），不要通用推广语 |
| 打开小程序 → 看到内容列表 | 20-30% | "要绑定账号，不敢""加载太慢" | 无需登录即可预览（基于公开主页数据生成预览），真正上架时再授权 |
| **看到估价 → 决定上架** | **30-40%** | **"价格太低，不值得""不确定安全性""觉得麻烦"** | **这是最大流失点。估价必须给到心理预期以上；安全保障信息要前置；流程极简** |
| 开始操作 → 完成上架 | 10-15% | "操作中遇到问题""临时有事放弃" | 流程不超过3步，2分钟内完成 |

**核心结论**：最大的流失发生在两个节点——"点不点开"（60-70%流失）和"上不上架"（30-40%流失）。两个节点的对策完全不同：

- "点不点开"靠话术和渠道信任（BD个人关系 > 官方私信 > 群发广告）
- "上不上架"靠产品体验（估价吸引力 + 安全感 + 操作简洁度）

**从100个触达对象到完成上架的漏斗**：
```
BD私信触达：        100人
点开看看：          30-40人
产生兴趣：          15-25人
决定上架：          10-15人
完成操作：          8-13人
```
**转化率约8-13%。** 这意味着要获得100个活跃创作者供给，需要BD触达约800-1200个创作者。按每人BD成本50元计，总成本约4-6万元。

## 4. "没有AI换人"时的使用场景

这是最关键的现实问题：如果不做AI换人（Phase 1明确排除），商家买到一条素材后，具体怎么用？

### 4.1 场景一：原素材直投

**操作**：商家拿到源文件，直接上传千川，建计划投放。不做任何修改。

| 维度 | 评估 |
|------|------|
| 可行性 | 技术上完全可行。千川支持第三方素材投放（需上传授权证明） |
| 去重风险 | 如果独占授权（只卖一个商家）→ 不存在去重问题。如果非独占（卖3-5个商家）→ 存在同质化挤压但可控（参见07_dedup研究） |
| 效果预期 | 这是跨账户迁移问题（H4）。预计eCPM衰减30-60%，但仍高于盲测的素材 |
| 合规性 | 需要肖像权授权+内容使用授权，平台可自动生成 |
| 付费意愿 | **最高**。商家要的就是"拿来就投"，最省事 |
| 核心问题 | **同脸多品牌**——如果同一个创作者的脸出现在多个品牌的千川广告中，用户刷到会产生不信任感。独占授权可解决，但市场规模缩小 |

**判断：这是Phase 1最直接可行的场景，尤其是独占授权模式下。** 但市场规模受限于"一条素材只卖一次"。

### 4.2 场景二：混剪加工

**操作**：商家拿到源文件后，由自己的剪辑团队进行混剪——保留部分画面，加入自家产品信息、品牌元素、新的配音和文案。

| 维度 | 评估 |
|------|------|
| 可行性 | 完全可行，且是投手最熟悉的工作方式（他们日常就在做混剪） |
| 去重风险 | 混剪改造>50%后，去重风险极低 |
| 效果预期 | **这里有一个悖论**：改造越深，去重风险越低，但跑量效果跟原素材的关联也越弱。如果改造到50%+，这条素材跟投手自己做一条新的区别不大 |
| 合规性 | 需要明确"可混剪"的授权条款。创作者的感受问题——如果被改得"很low"，创作者会不满（参见05_content_nature研究中的"被改low"风险） |
| 付费意愿 | **中等**。商家需要的是"混剪的原材料"而非"成品"。原材料的价值低于成品。定价：100-300元/条 |
| 核心问题 | 如果商家最终买的是"混剪原材料"，这个产品本质上是一个"带数据的素材库"——跟Pexels、Videvo等视频素材库的逻辑类似，只是多了"千川数据标签"。这个定位的天花板较低 |

**判断：混剪场景可行，但价值感弱。** 商家不会为"原材料"付高价。如果产品走这个方向，定位就退化成了"带投放数据标签的视频素材库"。

### 4.3 场景三：翻拍参考

**操作**：商家看到一条跑量素材的数据和完整视频后，不买素材本身，而是让自己的团队按相同结构翻拍一条。

| 维度 | 评估 |
|------|------|
| 可行性 | 完全可行，且是目前行业的主流做法（用有米有数看竞品→翻拍） |
| 去重风险 | 完全翻拍=全新素材，零去重风险 |
| 效果预期 | 翻拍的效果衰减严重。相同脚本换人拍，效果通常衰减50-80%。因为"跑量基因"不仅在脚本结构中，也在表演细节、节奏感、微表情等不可言说的因素中 |
| 合规性 | 翻拍是否侵权？如果只学结构不用原素材，法律上很难判定侵权。但如果结构过于相似，有"洗稿"之嫌 |
| 付费意愿 | **最低**。这就是现在有米有数/蝉妈妈在做的事——看竞品素材找灵感。商家愿意为"看到跑量素材+数据"付费（工具年费2-5万），但不愿意为单条素材的"翻拍参考权"付费 |
| 核心问题 | **如果商家的真实需求只是"看到跑量素材和数据，然后自己翻拍"，那我们做的就不是"内容交易市场"，而是"带数据的蝉妈妈"** |

**判断：这是一个非常真实的风险。** 如果H4验证结果显示跨账户效果衰减>50%（原素材直投不靠谱），商家的理性选择就是"看数据→自己翻拍"而不是"买素材→直接投"。此时产品退化为数据工具而非交易市场。

### 4.4 场景四：AI换人（未来加入）

**操作**：商家购买"已验证跑量的内容模板"，通过AI技术将创作者的脸和声音替换为模特或虚拟人，保留原始素材的节奏、脚本结构和表演动作。

| 维度 | 评估 |
|------|------|
| 可行性 | 2026年技术：正脸口播8/10真实度，复杂场景5/10。还不够成熟 |
| 去重风险 | 换脸后画面变化>50%，大概率绕过去重 |
| 效果预期 | 如果AI换人质量够高，理论上可以保留原素材80%+的"跑量基因"（因为节奏、脚本、动作都不变，只变了脸）。但用户对AI面孔的识别力在快速提升 |
| 合规性 | 深度合成标识要求、肖像权问题。法律灰色地带 |
| 付费意愿 | **高**。如果实现了"验证过的模板+换脸=独一无二的高概率跑量素材"，商家愿付1000-3000元/条 |
| 核心问题 | 技术不成熟、法律不明确、平台态度不确定 |

**判断：AI换人是唯一一个能让"内容交易市场"真正成为独特产品形态（而不是退化为数据工具或素材库）的场景。** 但Phase 1不做是正确的。需要在Phase 2提前布局。

### 4.5 四种场景的关键判断

**如果没有AI换人，这个产品在Phase 1能做什么？**

实际上只有两种可行的产品形态：

**形态A："带数据的素材直购"（原素材直投，独占授权）**
- 商家买一条素材，直接投千川
- 优势：简单、直接、付费意愿高
- 劣势：独占授权市场小，跨账户效果衰减是致命风险
- 本质：内容交易市场

**形态B："带数据的素材灵感推荐"（翻拍参考+部分混剪）**
- 商家看素材数据，找灵感，自己翻拍或混剪
- 优势：不受去重/跨账户迁移限制，需求基础更广
- 劣势：付费意愿低（年费工具逻辑，不是单次交易逻辑），跟有米有数/蝉妈妈直接竞争
- 本质：带数据的蝉妈妈

**H4验证结果决定了走哪条路：**
- H4成立（衰减<40%）→ 形态A，做内容交易市场
- H4不成立（衰减>50%）→ 形态B，做数据工具
- 两者之间（衰减40-50%）→ 两种形态并行，看哪个跑出来

**一个残酷的可能性：如果最终商家买的是"灵感/模板"，这本质上就是"带数据的蝉妈妈"而不是"内容交易市场"。** 两者的商业天花板差5-10倍。内容交易市场可以做到年GMV数亿（参考图片素材库的规模），而数据工具的天花板是年收入几千万（参考有米有数的规模）。

## 5. 竞品的真实使用体验

### 5.1 有米有数

**投手真实使用方式**：
- 每周花2-3小时，打开"千川热门素材"板块
- 按品类筛选，看"投放天数>3天"且"预估消耗>10万"的素材
- 打开视频看——主要看前3秒hook方式、产品展示角度、整体结构
- 截图/录屏，发给剪辑说"做这种风格的"
- **从来不看有米有数给的"数据分析"**——投手知道那些数据是估算的，不是真实投放数据

**用户习惯关键点**：
- 是"灵感获取工具"，不是"素材获取工具"
- 投手只"看"不"买"——因为没有购买选项
- 使用频率：每周1-2次，每次20-40分钟

**我们与有米有数的差异**：
- 有米有数：看到"别人在投什么"→ 自己翻拍
- 我们（理想状态）：看到"别人投了效果是什么"→ 直接购买投放
- **差异本质：从"猜测"到"验证"，从"看"到"买"**
- 但如果H4不成立，我们跟有米有数的差异就只剩"数据更准"——这是有价值的，但不够颠覆性

### 5.2 蝉妈妈

**投手真实使用方式**：
- 主要用来看"达人带货排行"和"热门商品"
- 不太用来看素材——蝉妈妈的素材分析功能不如有米有数
- 更多被运营/选品人员使用，投手自己用得少

**用户习惯关键点**：
- 定位是"电商数据工具"而非"素材工具"
- 跟我们的产品重叠度有限

### 5.3 巨量创意

**投手真实使用方式**：
- 两个功能最常用：
  1. "创意排行"——看行业跑量素材的概览（视频缩略图+标签，不能下载）
  2. "即创"——紧急情况下用模板快速生成一条应急素材
- "创意排行"的问题：只能看到视频外观和基础标签，看不到任何效果数据（连预估都没有）
- "即创"的问题：模板同质化极严重，几十万人用同样的模板，效果在快速递减

**用户习惯关键点**：
- 是千川后台内置功能，使用零门槛（不用打开新工具）
- 这个"零门槛"优势极大——任何需要跳出千川后台的工具都面临这个竞争劣势

**我们与巨量创意的差异**：
- 巨量创意：提供模板和AI生成工具，方便自己做
- 我们：提供已验证的真人素材，直接买
- **差异本质：AI模板 vs 真人验证素材。只要真人素材的跑量率>AI素材，差异就成立**

### 5.4 素材代拍

**投手真实使用方式**：
- 腰部商家的主要外部素材来源
- 典型流程：投手写需求brief → 发给代拍公司 → 等7-15天 → 收到3-10条素材 → 投千川测试
- 成本：300-3000元/条，取决于品类和质量要求

**用户习惯关键点**：
- 最大的痛点不是"贵"，是"等"——7-15天的交付周期，在千川的节奏下太慢了
- 次大的痛点是"不确定"——交付的素材没有任何效果背书，100%靠运气

**我们与代拍的差异**：
- 代拍：花钱→等→拿到未验证素材→自己测
- 我们：花钱→立刻拿到→已有数据验证的素材→直接投
- **差异本质：从"等+赌"到"即时+确定"**
- 这是最清晰的差异化，也是投手最能感知的价值

### 5.5 精确差异化总结

| 竞品/替代 | 他们提供的 | 我们提供的 | 差异点 | 差异强度 |
|-----------|-----------|-----------|--------|---------|
| 有米有数 | 竞品在投什么（模糊的预估数据） | 具体素材的真实投放效果数据 + 可购买 | 数据精度 + 可交易 | 中（如果只做数据工具）/ 强（如果做交易） |
| 巨量创意 | AI模板 + 行业概览 | 真人已验证素材 | 真人 vs AI，验证 vs 模板 | 强（只要真人>AI） |
| 素材代拍 | 7-15天交付未验证素材 | 即时获取已验证素材 | 速度 + 确定性 | **最强** |
| 星图达人 | 达人定制（贵+慢+不确定） | 闲置素材采购（便宜+快+有数据） | 性价比 + 速度 | 强 |
| 盗用/搬运 | 免费但违法 | 付费但合法+有数据 | 合规性 + 数据 | 弱（盗用成本太低，需要平台审核收紧才能逼转） |

**最大的竞争优势在"vs素材代拍"这条线上**——速度+确定性。这应该是核心卖点话术。

**最大的竞争劣势在"vs盗用"这条线上**——只要盗用成本几乎为零且风险很低，很多商家没动力付费买正版。这需要平台审核收紧配合（参见05_content_nature研究）。

## 6. "啊哈时刻"分析

### 6.1 商家端的啊哈时刻

**理论上的啊哈时刻**："我在平台买了一条素材，投千川跑出了ROAS 2.0+"

**问题**：从购买到看到投放结果，需要多久？
- 购买 → 上传千川 → 过审（20分钟-2小时）→ 建计划 → 开投 → 冷启动（6-24小时）→ 看到初步数据
- **最快：当天晚上看到数据（如果早上买的话）**
- **最慢：3-5天才能判断这条素材是否真的跑起来了**

**如果需要等3-5天才能到达啊哈时刻，中间会发生什么？**
- 第1天：买了，上传了，开投了，有点期待
- 第2天：数据一般般，冷启动期，看不出好坏。开始怀疑"是不是白花钱了"
- 第3天：如果数据上升 → 开始兴奋。如果数据还是一般 → 基本放弃了，认为"跟自己做的差不多"
- 第5天：如果一直没跑起来 → 负面结论已经形成，不会再买

**更现实的啊哈时刻设计**：不能等投放结果。应该把啊哈时刻前移到**购买环节本身**。

**前移方案：让商家在购买的瞬间就感受到价值**
- 啊哈时刻v2：**"我搜了我的品类，看到了10条带千川真实数据的素材，包括ROAS、消耗额、投放天数——这些信息我在有米有数上看不到！"**
- 这个啊哈时刻在进入产品后30秒内就能到达
- 即使商家最终没买素材，"能看到真实投放数据"本身就是价值——这个价值支撑了"数据工具"的定位

**分层设计**：
| 时间点 | 啊哈时刻 | 触发条件 | 价值感知 |
|--------|---------|---------|---------|
| 进入产品后30秒 | "能看到品类内的真实千川投放数据" | 搜索品类后展示素材列表+数据 | 信息价值（"这个数据很有用"） |
| 购买后当天 | "拿到源文件，上传千川，建了计划" | 文件下载+上传成功 | 效率价值（"这么快就上线了"） |
| 购买后1-3天 | "这条素材在我账户上跑起来了" | 千川数据回正 | **终极价值（"花的钱值了"）** |
| 购买后1周 | "这条素材累计消耗了5万+，ROAS 2.0+" | 持续跑量 | 复购动力形成 |

### 6.2 创作者端的啊哈时刻

**理论上的啊哈时刻**："我的旧视频卖出去了，收到了钱"

**问题**：从上架到第一笔收入，需要多久？
- 如果市场还没有足够的商家需求 → 可能等1-4周（甚至更久）
- 如果平台主动撮合（BD手动匹配）→ 可能3-7天

**等待1周以上才有收入的后果**：
- 第1天：上架了，有点期待
- 第3天：还没卖出去，开始忘了这件事
- 第7天：完全忘了。如果此时收到"你的内容卖出去了"的通知 → 惊喜 → 但可能已经不记得这个平台了
- 第14天：如果还没卖出去 → 永远不会再打开

**更现实的啊哈时刻设计**：不能等第一笔交易。应该把啊哈时刻前移到**上架的瞬间**。

**前移方案**：
- **上架即给激励金**：创作者上架5条内容，立刻到账20元"品鉴金"（不是"补贴"，是"谢谢你信任我们"）
- 这笔钱极小（平台成本低），但作用极大：
  1. 创作者第一次体验到"这个平台真的会给我钱"——信任建立
  2. 他截图发朋友圈的概率大幅提高（"上架5条视频就得了20块"比"注册了一个平台"有传播力得多）
  3. 把啊哈时刻从"等N天后第一笔交易"缩短到"上架后1分钟内"

**分层设计**：
| 时间点 | 啊哈时刻 | 触发条件 | 价值感知 |
|--------|---------|---------|---------|
| 上架后1分钟 | "上架就到了20块品鉴金" | 上架5条以上 | 信任建立（"这平台是真的给钱"） |
| 上架后3-7天 | "你的内容被XX商家购买了" | 第一笔交易通知+到账 | **终极价值（"躺着赚钱"）** |
| 上架后1个月 | "本月收入：800元" | 月度收入汇总 | 持续价值（"每月多赚一顿火锅钱"） |

### 6.3 啊哈时刻的核心风险

**商家端**：如果首次购买的素材跑不起来（概率50-70%），啊哈时刻永远不会到达。这一个失败体验可能抹杀所有的产品优势。

**对策**：
- 前50个商家不走自助购买。BD根据商家的品类和账户情况，手动推荐最有可能跑量的素材
- 提供"首单保障"：首次购买如果投放3天ROAS<1.0，全额退款
- 把首次成功率从自然概率（30-40%）人为提高到60%+

**创作者端**：如果上架1个月都没有交易，创作者会永久流失。

**对策**：
- 冷启动期必须有保底收购机制——平台自己花钱买10-20条种子素材，确保第一批创作者都有成交
- 成本：10-20条 × 300元 = 3000-6000元。这不是补贴，是冷启动投资

---

## 7. 最终判断

### 7.1 真实定位应该是什么？

不是"内容交易市场"。至少Phase 1不是。

**Phase 1的真实定位：带千川投放数据的素材发现与直购工具。**

为什么不是"交易市场"：
- 交易市场意味着双边平台、流动性、自动撮合——Phase 1做不到，也不需要
- 投手不会"逛市场"，他们会"搜索+筛选+购买"——这是工具行为，不是逛集市行为
- 创作者不会"经营店铺"，他们会"上架一次然后忘掉"——这是被动收入，不是开店

**更精准的定位取决于H4验证结果**：

| H4结果 | 产品定位 | 商业模式 | 天花板 |
|--------|---------|---------|--------|
| 衰减<30% | **千川效果素材直购平台** | 按条买断，500-2000元/条 | 年GMV数亿级 |
| 衰减30-50% | **带数据的素材推荐+直购工具** | 混合模式：部分直购+部分当灵感参考 | 年GMV数千万级 |
| 衰减>50% | **千川素材效果数据库（SaaS工具）** | 年费5-15万/客户 | 年收入数千万级 |

### 7.2 用户愿意为之付费的核心价值

**一句话：帮投手在每天10点到11点半的"找素材方向"环节，从"看别人在投什么然后猜"升级为"看到别人投了效果是什么然后直接买"。**

拆开说：
- 投手付费买的不是"视频文件"——视频文件到处都有
- 投手付费买的不是"创意灵感"——有米有数/巨量创意已经在提供
- 投手付费买的是**"千川跑量的确定性"**——一条素材在别的账户跑出ROAS 2.5，这个信息本身就值几百块
- 更精确地说：投手买的是**"降低测试成本的保险"**——与其花2000块盲测4条素材（全部扑街概率70%），不如花500块买1条有数据背书的素材（扑街概率降到40-50%）

### 7.3 如果只能做一件事来证明产品价值

**做10笔手动撮合交易，跟踪跨账户投放效果。**

具体操作：
1. 找5个长尾创作者，每人拿2条近期在千川跑量过的素材（共10条）
2. 记录每条素材在原账户的eCPM、CTR、CVR、ROAS
3. 找5个腰部商家（品类匹配），每人分配2条素材
4. 商家在自己的千川账户投放这些素材，各投2000元预算
5. 记录每条素材在新账户的eCPM、CTR、CVR、ROAS
6. 对比原始数据和迁移后数据

**总成本**：
- 向创作者采购10条素材：10 × 500元 = 5,000元
- 商家测试投放预算：10 × 2,000元 = 20,000元（可由项目方补贴或商家自承）
- 人力成本：2人 × 2周 = 若干
- **总计约3-5万元**

**这笔钱能回答的问题**：
1. H4：跨账户eCPM衰减率是多少？
2. 商家对"带数据素材"的真实付费意愿有多强？
3. 创作者对出售归档内容的接受度和底价是什么？
4. 整个链路的操作摩擦在哪里？

**如果eCPM衰减<40%**：全力推进内容交易市场
**如果eCPM衰减40-60%**：转向"数据+素材推荐工具"的混合定位
**如果eCPM衰减>60%**：放弃交易市场，转B计划（千川素材效果预测SaaS）

**不做别的。不做产品开发，不做品牌建设，不做创作者社群运营。先花3-5万块钱回答这一个问题。** 这一个问题的答案决定了后面几百万的投入方向是否正确。

## 附：几个逆向思考

### 思考一：如果这个产品最终成了"带数据的蝉妈妈"，是坏事吗？

不一定。蝉妈妈年收入数亿，是一门好生意。如果我们能做一个"比蝉妈妈数据更准（因为有真实投放数据）、且能直接购买素材的工具"，即使退化为SaaS，也是一个有价值的产品。只是天花板不如"交易市场"高。

### 思考二：500元/条的定价能撑起多大的市场？

简单算：
- 目标用户：年投放100-500万的腰部商家，全国约5-8万家
- 可渗透率：3-5%（第二年），即1500-4000家
- 月购买频次：5-15条
- 客单价：500-800元
- 月GMV：375万-4800万
- 平台抽佣20%：月收入75万-960万
- **年收入：900万-1.15亿**

乐观场景下天花板约1亿/年。中性场景约3000-5000万/年。这是一个"不大不小"的生意——不足以支撑独角兽叙事，但足以成为一个盈利的垂直工具。

### 思考三：最让我担心的一件事

不是H4，不是平台封杀，不是AIGC替代。

是**"投手的使用习惯太固化了"**。

投手每天的工作流已经形成：千川后台→有米有数→微信群→剪辑团队。插入一个新工具到这个流程中，需要改变的不是"认知"，而是"肌肉记忆"。认知可以通过教育改变，肌肉记忆需要反复使用才能建立。

如果产品不是千川后台的原生功能，而是一个独立的小程序/网站，投手每次使用都需要"记起来有这个工具→打开→搜索→购买"——这个动作链太长了。大部分人在第一步"记起来"就断了。

**所以，嵌入千川生态不是"更好的选择"，是"唯一的选择"。** 独立产品在这个市场里活不下来。
