## 1. 实验目标

### 1.1 核心假设

**H4：** 一条在原始账户上"跑量成功"的素材，授权给新账户投放后，跑量成功率不会衰减超过 50%。

精确表述：

```
设 P_origin 为素材在原始账户的跑量成功率（历史已观测）
设 P_transfer 为同一素材在新账户的跑量成功率（实验待观测）

H4通过条件：P_transfer / P_origin >= 0.50
即衰减率 = 1 - P_transfer / P_origin <= 0.50
```

### 1.2 核心指标

| 指标 | 定义 | 数据来源 |
|------|------|----------|
| **主指标：跑量成功率** | 单条素材在千川投放中，7天累计消耗 >= 5000 元即判定为"跑量成功" | 千川后台导出 |
| 辅助指标1：eCPM | 千次展示成本，= CTR x CVR x 客单价 x 1000 | 千川后台 |
| 辅助指标2：3秒完播率 | 前3秒留存比例，衡量素材本身的吸引力 | 千川后台 |
| 辅助指标3：CTR | 点击率，衡量素材封面/前几帧的吸引力 | 千川后台 |
| 辅助指标4：CVR | 转化率，衡量素材的说服力和产品匹配度 | 千川后台 |
| 辅助指标5：CPM | 千次展示出价成本，衡量系统对素材的流量分配意愿 | 千川后台 |

### 1.3 通过/失败判定

| 结论 | 条件 | 后续动作 |
|------|------|----------|
| **通过（绿灯）** | 跨账户跑量成功率 >= 原始的 50%，且统计检验 p < 0.05 | 项目继续，进入产品开发阶段 |
| **有条件通过（黄灯）** | 跨账户跑量成功率在原始的 35%-50% 之间 | 项目继续，但需收窄匹配条件（同品类+同客单价段+相似受众画像），并调整商业预期 |
| **失败（红灯）** | 跨账户跑量成功率 < 原始的 35%，或与随机自制素材无统计显著差异 | 启动B计划评估（内容数据SaaS），第4周末Gate 1决策时止损或转型 |
| **不确定** | 样本量不足以在 p < 0.10 水平下做出判断 | 延长实验2周，同时降低MDE重新评估 |

## 2. 实验设计

### 2.1 样本量计算（功效分析）

**参数设定：**

| 参数 | 取值 | 依据 |
|------|------|------|
| 基线跑量成功率 P_baseline | 10% | 投手经验：商家自制素材约10条跑1条 |
| 预期采购素材跑量成功率 P_treatment | 20% | 算法研究预判：top 20%素材可提升至15%-25%，取中位数 |
| 检验类型 | 单侧 Z 检验（比例检验） | 我们关心的是"采购素材是否优于自制"，单侧即可 |
| 显著性水平 alpha | 0.05 | 标准选择 |
| 统计功效 1 - beta | 0.80 | 标准选择 |
| 最小可检测效应量 MDE | 10个百分点（10% -> 20%） | 即跑量成功率翻倍，这是商业上有意义的最小改善 |

**样本量公式（两独立比例的单侧检验）：**

```
n = [ Z_alpha * sqrt(P_bar*(1-P_bar)*2) + Z_beta * sqrt(P_baseline*(1-P_baseline) + P_treatment*(1-P_treatment)) ]^2
    / (P_treatment - P_baseline)^2

其中：
  P_bar = (P_baseline + P_treatment) / 2 = 0.15
  Z_0.05 = 1.645
  Z_0.20 = 0.842

n = [1.645 * sqrt(0.15*0.85*2) + 0.842 * sqrt(0.10*0.90 + 0.20*0.80)]^2 / (0.10)^2
  = [1.645 * 0.5050 + 0.842 * 0.5099]^2 / 0.01
  = [0.8307 + 0.4293]^2 / 0.01
  = [1.2600]^2 / 0.01
  = 1.5876 / 0.01
  = 158.8

每组约需 159 次投放测试
```

**但这是理想情况。考虑到我们的实际约束（手动撮合阶段），调整如下：**

| 场景 | 每组样本量 | 总投放次数 | 可检测的MDE | 说明 |
|------|-----------|-----------|-------------|------|
| 理想方案 | 159 | 318 | 10pp (10%->20%) | 需要约30条素材 x 每条10次投放，不现实 |
| 务实方案（推荐） | 60 | 120 | 15pp (10%->25%) | 需要约15条素材 x 每条在4个新账户投放 |
| 最小方案 | 30 | 60 | 20pp (10%->30%) | 需要约10条素材 x 每条在3个新账户投放，功效降至0.65 |

**推荐采用务实方案：**
- 选 15 条已验证跑量素材
- 每条分配给 4 个不同的新商家账户投放
- 共产生 60 次"采购素材投放"观测
- 同时要求每个商家在同一周期内投放至少 1 条自制素材作为对照
- 共产生 60 次"自制素材投放"观测（每个商家约 3-4 条自制素材参与对照）

### 2.2 对照组设计

采用 **"配对内对照"设计（Within-Merchant Paired Control）**：

```
对于每个参与实验的商家 j：
  实验组：商家 j 投放采购素材 i -> 记录 outcome_{ij}
  对照组：商家 j 在同一周期投放自制素材 -> 记录 outcome_{j,self}

配对比较：delta_j = outcome_{ij} - outcome_{j,self}
```

**为什么用配对设计而不是独立两组：**
- 消除商家间差异（账户权重、投放能力、品类差异）
- 同一商家、同一时段投放，竞价环境基本一致
- 所需样本量更小（约为独立两组的 60-70%）

**对照组的具体操作：**

| 要素 | 对照组（自制素材） | 实验组（采购素材） |
|------|-------------------|-------------------|
| 投放主体 | 同一商家账户 | 同一商家账户 |
| 投放时段 | 同一周的同一时段 | 同一周的同一时段 |
| 投放预算 | 相同 | 相同 |
| 出价策略 | 相同 | 相同 |
| 人群定向 | 相同 | 相同 |
| 素材来源 | 商家自行制作 | 从平台采购的已验证素材 |
| 素材数量 | 每个商家至少1条（建议3条取中位数） | 每个商家至少2条 |

### 2.3 分组策略

#### 按品类分层（必须）

| 品类 | 目标素材数 | 目标商家数 | 选择理由 |
|------|-----------|-----------|----------|
| 美妆/护肤 | 5条 | 6-8个商家 | 客单价高，投手需求旺盛，素材模式化强 |
| 食品/零食 | 5条 | 6-8个商家 | 市场大，复购驱动，素材风格多样 |
| 3C/数码 | 5条 | 4-6个商家 | 客单价高，功能展示为主，迁移性可能更好 |

#### 按商家规模分层（尽量）

| 商家规模 | 定义 | 目标数量 | 预期差异 |
|----------|------|----------|----------|
| 腰部商家 | 月千川消耗 5-30万 | 10-12个 | 核心目标客群，实验权重最高 |
| 小型商家 | 月千川消耗 1-5万 | 4-6个 | 预算有限但试错需求强 |

不纳入头部商家（月消耗>30万），原因：
- 头部商家通常有专业内容团队，采购外部素材的动力弱
- 头部账户权重高，可能天然跑量好，导致实验结果偏高

#### 按素材类型标注（分析用）

每条素材标注以下类型标签（非分组依据，仅用于亚组分析）：

- 口播型 / 剧情型 / 混剪型 / 产品展示型
- 有真人出镜 / 无真人出镜
- 时长：<15s / 15-30s / 30-60s / >60s

### 2.4 控制变量

为保证实验内部效度，以下变量必须在实验组和对照组间保持一致：

| 控制变量 | 标准化方法 | 执行人 |
|----------|-----------|--------|
| **投放预算** | 每条素材统一测试预算 500 元/天，连续投放 7 天，共计 3500 元/条 | 投手/商家 |
| **出价策略** | 统一使用"控成本投放"（OCPM），目标成本按品类设定（见下表） | 投手/商家 |
| **人群定向** | 统一使用"系统智能推荐"（即不做人工定向限制），让算法自行分配流量 | 投手/商家 |
| **投放时段** | 全天投放，不做时段限制 | 投手/商家 |
| **创意组合** | 每条素材单独建计划，不与其他素材混合投放 | 投手/商家 |
| **投放同期性** | 同一商家的实验组和对照组素材必须在同一周内上线 | 运营协调 |

**各品类目标成本参考：**

| 品类 | 目标转化成本（参考） | 说明 |
|------|---------------------|------|
| 美妆/护肤 | 50-80 元/单 | 按行业均值设定 |
| 食品/零食 | 20-40 元/单 | 客单价低，转化成本低 |
| 3C/数码 | 80-150 元/单 | 客单价高，转化成本高 |

## 4. 分析方法

### 4.1 主分析：配对比例检验

**方法：McNemar 检验 + 配对比例差的置信区间**

每个商家产生一个配对观测：
```
商家 j 的采购素材跑量结果 X_j ∈ {0, 1}
商家 j 的自制素材跑量结果 Y_j ∈ {0, 1}

如果商家 j 有多条采购/自制素材，取各自的跑量成功率：
  P_purchase_j = (跑量成功的采购素材数) / (该商家采购素材总数)
  P_self_j = (跑量成功的自制素材数) / (该商家自制素材总数)
```

**主检验：**
```
H0: P_purchase - P_self <= 0  (采购素材不优于自制)
H1: P_purchase - P_self > 0   (采购素材优于自制)

检验统计量：配对 t 检验（对每个商家的 delta_j = P_purchase_j - P_self_j）
  t = mean(delta_j) / (sd(delta_j) / sqrt(n_merchants))

判定：p < 0.05 拒绝 H0
```

**效果量计算：**
```
跑量成功率比值 = mean(P_purchase_j) / mean(P_self_j)
跑量成功率差值 = mean(P_purchase_j) - mean(P_self_j)
95% 置信区间：mean(delta_j) +/- t_0.025 * se(delta_j)
```

**备选方法（如果配对数据不完整）：**

使用独立两样本的 Fisher 精确检验：
```
构建 2x2 列联表：
            跑量成功    跑量失败
采购素材      a           b
自制素材      c           d

Fisher 精确检验的 p 值
优势比 OR = (a*d) / (b*c)，及其 95% 置信区间
```

### 4.2 辅助分析：连续指标比较

对 eCPM、CTR、CVR、CPA 等连续指标：

```
配对 Wilcoxon 符号秩检验（非参数，不假设正态分布）：
  对每个商家 j：delta_eCPM_j = eCPM_purchase_j - eCPM_self_j
  检验 delta_eCPM_j 的中位数是否 > 0

效果量：Hodges-Lehmann 估计量（中位数差值的置信区间）
```

**eCPM 衰减率计算：**
```
对每条采购素材 i 在新商家 j 的投放：
  衰减率_ij = 1 - (eCPM_new_ij / eCPM_original_i)

报告：
  衰减率的中位数和四分位距
  衰减率 <= 30% 的比例
  衰减率 <= 50% 的比例
```

### 4.3 亚组分析

| 亚组维度 | 比较内容 | 预期 |
|----------|----------|------|
| 品类 | 美妆 vs 食品 vs 3C 的衰减率差异 | 产品展示型（3C）迁移性可能好于口播型（美妆） |
| 商家规模 | 腰部(5-30万) vs 小型(1-5万) | 腰部商家账户权重更高，可能跑量效果更好 |
| 素材类型 | 口播 vs 剧情 vs 混剪 vs 产品展示 | 产品展示型依赖品牌露出少，迁移性可能更好 |
| 素材时效 | 距峰值 <14天 vs 14-30天 | 越新鲜的素材衰减可能越小 |
| 真人出镜 | 有真人 vs 无真人 | 无真人素材没有"换脸"问题，迁移性可能更好 |
| 品类匹配度 | 原始品类=买方品类 vs 不同 | 同品类迁移预期显著优于跨品类 |

**亚组分析注意事项：**
- 亚组分析为探索性分析，不做多重检验校正
- 每个亚组样本量可能不足以单独达到统计显著，主要关注效果方向和效应量
- 对结果有临床意义（效应量大）但统计不显著的亚组，标记为"需要后续大样本验证"

### 4.4 敏感性分析

| 分析 | 目的 | 方法 |
|------|------|------|
| 跑量阈值敏感性 | 检验结论是否依赖于"5000元"这个跑量定义 | 分别用 3000 元和 10000 元重新计算跑量成功率和衰减率 |
| 异常值敏感性 | 检验结论是否被极端值驱动 | 去除 eCPM 最高和最低 10% 的观测后重新分析 |
| 时间窗口敏感性 | 检验 7 天是否足够评估 | 对比 5 天 vs 7 天的跑量判定结果是否一致 |
| 首日消耗敏感性 | 首日冷启动效应是否扭曲结果 | 去除第 1 天数据，仅用第 2-7 天重新分析 |
| 缺失数据敏感性 | 如果部分商家中途退出怎么办 | 最坏情况分析：所有缺失数据视为"跑量失败"，重新计算 |

## 5. 判定标准

### 5.1 定量门槛

**主判定：跑量成功率对比**

| 指标 | 通过（绿灯） | 有条件通过（黄灯） | 失败（红灯） |
|------|-------------|-------------------|-------------|
| 采购素材跑量成功率 | >= 15%（绝对值） | 10%-15% | < 10% |
| 相对自制素材的提升 | >= 1.5 倍 | 1.0-1.5 倍 | <= 1.0 倍 |
| 跨账户eCPM衰减率中位数 | <= 30% | 30%-50% | > 50% |
| 统计显著性（主检验） | p < 0.05 | p < 0.10 | p >= 0.10 |

**辅助判定：**

| 指标 | 参考标准 | 说明 |
|------|----------|------|
| 采购素材 CTR vs 自制素材 CTR | 采购 >= 自制的 0.8 倍 | CTR 衰减不超过 20% |
| 采购素材 3秒完播率 vs 原始 | 采购 >= 原始的 0.7 倍 | 素材内在吸引力保留度 |
| 7日内至少有1天消耗>1000元 | >= 60% 的采购素材满足 | 系统至少给了初始流量 |

### 5.2 最小可检测效应量（MDE）

| 方案 | 每组样本量 | MDE（alpha=0.05, power=0.80） | 含义 |
|------|-----------|------|------|
| 务实方案（推荐） | 60 | 15 百分点 | 能检测出跑量率从 10% 提升到 25% 的效果 |
| 最小方案 | 30 | 20 百分点 | 能检测出跑量率从 10% 提升到 30% 的效果 |

如果实际效应小于 MDE，实验将无法检出——这意味着：
- 如果采购素材确实只比自制好一点点（如从 10% 提升到 15%），我们的样本量可能不够
- 此时结论为"不确定"，而非"失败"

### 5.3 样本不足的应对方案

| 情况 | 判断标准 | 应对措施 |
|------|----------|----------|
| 商家招募不足 | 第2周末参与商家 < 12 个 | 扩大招募范围，增加激励（免费送3条素材测试） |
| 投放数据不足 | 第3周末完成投放的交易 < 20 笔 | 延长实验至第6周，同时降低MDE至 20pp |
| 数据回传不足 | 投放数据回传率 < 60% | 运营1v1跟进，派人远程协助截图录数据 |
| 对照组缺失 | 部分商家没有同期自制素材对照 | 使用该商家近30天内的自制素材历史数据作为历史对照 |

#### 商家投放测试SOP v1.0

**目的：** 测试采购素材在您账户上的投放效果，与您的自制素材做对比。

**Step 3：设置投放计划（Day 0-1）**

  - 3C/数码：80-150 元/单

- 创意分类和标签：按您日常习惯设置

- 选择您最近7天内新制作的 1-3 条自制素材

- 命名规则：`[测试]自制素材-编号YYY`

- 每天上午 10:00 前，截图千川后台数据面板

**Step 6：投放结束（Day 7）**
- 第7天结束后暂停投放计划
- 导出7日汇总数据截图
- 回答运营的简短问卷（5分钟）

- 测试期间请勿调整出价、定向或预算

## 附录B：分析代码框架（Python）

以下代码框架供数据分析师在实验结束后使用：

```python
import pandas as pd
import numpy as np
from scipy import stats

# =============================
# 1. Load and prepare data
# =============================
summary = pd.read_csv('7day_summary.csv')

purchased = summary[summary['is_purchased'] == 1]
self_made = summary[summary['is_purchased'] == 0]

# =============================
# 2. Primary analysis: run success rate comparison
# =============================
n_purchased = len(purchased)
n_self = len(self_made)
success_purchased = purchased['run_success'].sum()
success_self = self_made['run_success'].sum()

rate_purchased = success_purchased / n_purchased
rate_self = success_self / n_self

print(f"Purchased material run rate: {rate_purchased:.1%} ({success_purchased}/{n_purchased})")
print(f"Self-made material run rate: {rate_self:.1%} ({success_self}/{n_self})")
print(f"Lift: {rate_purchased / rate_self:.2f}x")

# Fisher exact test (one-sided)
table = [[success_purchased, n_purchased - success_purchased],
         [success_self, n_self - success_self]]
odds_ratio, p_value = stats.fisher_exact(table, alternative='greater')
print(f"Fisher exact test p-value: {p_value:.4f}")
print(f"Odds ratio: {odds_ratio:.2f}")

# =============================
# 3. eCPM decay analysis
# =============================
# Merge with original eCPM from material table
materials = pd.read_csv('material_info.csv')
purchased_with_original = purchased.merge(materials[['material_id', 'original_ecpm']], on='material_id')
purchased_with_original['ecpm_decay'] = 1 - (purchased_with_original['avg_ecpm'] / purchased_with_original['original_ecpm'])

median_decay = purchased_with_original['ecpm_decay'].median()
pct_under_30 = (purchased_with_original['ecpm_decay'] <= 0.30).mean()
pct_under_50 = (purchased_with_original['ecpm_decay'] <= 0.50).mean()

print(f"\neCPM decay median: {median_decay:.1%}")
print(f"Proportion with decay <= 30%: {pct_under_30:.1%}")
print(f"Proportion with decay <= 50%: {pct_under_50:.1%}")

# =============================
# 4. Paired analysis (within-merchant)
# =============================
merchant_stats = summary.groupby(['merchant_id', 'is_purchased'])['run_success'].mean().unstack()
merchant_stats.columns = ['self_rate', 'purchased_rate']
merchant_stats['delta'] = merchant_stats['purchased_rate'] - merchant_stats['self_rate']

t_stat, p_paired = stats.ttest_1samp(merchant_stats['delta'].dropna(), 0)
# Convert to one-sided
p_one_sided = p_paired / 2 if t_stat > 0 else 1 - p_paired / 2

print(f"\nPaired analysis:")
print(f"Mean delta (purchased - self): {merchant_stats['delta'].mean():.3f}")
print(f"Paired t-test p-value (one-sided): {p_one_sided:.4f}")

# =============================
# 5. Subgroup analysis
# =============================
for col in ['category', 'material_type', 'has_human_face']:
    print(f"\n--- Subgroup: {col} ---")
    for val, grp in purchased_with_original.groupby(col):
        decay = grp['ecpm_decay'].median()
        run_rate = grp['run_success'].mean()
        print(f"  {val}: median decay={decay:.1%}, run rate={run_rate:.1%}, n={len(grp)}")

# =============================
# 6. Sensitivity: threshold analysis
# =============================
for threshold_name, threshold_col in [('3000', 'run_success_3k'), ('5000', 'run_success'), ('10000', 'run_success_10k')]:
    rate_p = purchased[threshold_col].mean()
    rate_s = self_made[threshold_col].mean()
    print(f"\nThreshold {threshold_name}: purchased={rate_p:.1%}, self={rate_s:.1%}, lift={rate_p/max(rate_s,0.001):.2f}x")
```
